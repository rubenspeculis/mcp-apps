# @mcpapps/client-core

The renderer-agnostic runtime that runs inside the `ui://` iframe. It owns the only `postMessage` JSON-RPC peer in the framework.

## API

```ts
import { createHostBridge, mountApp } from "@mcpapps/client-core";

const bridge = createHostBridge();          // connects to window.parent
bridge.onToolResult((env) => { /* … */ });
const out = await bridge.callTool("get_weather", { city: "London" });
await bridge.initialize();                  // ui/initialize → initialized handshake
```

- `createHostBridge(options?)` → `HostBridge` (see [the bridge](/concepts/the-bridge)).
- `mountApp(adapter, options?)` — bootstrap: build the bridge, mount the renderer adapter, run `initialize()`, and report size on resize (`ui/notifications/size-changed`) so flexible host iframes don't collapse to 0px.
- `RendererAdapter` — the interface each renderer implements (`mount(root, bridge)`).
- `createMockTransportPair()` / `createPostMessageTransport()` — transports; the mock pair powers tests and the emulator.

::note
You rarely import this directly — `@mcpapps/vue` and `@mcpapps/flutter` sit on top of it. It's the reason every renderer behaves identically against the host.
::
