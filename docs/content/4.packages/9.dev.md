# @mcpapps/dev

The local development surface: a host emulator, a Node dev server, and a Cloudflare tunnel.

## API

```ts
import { serveEmulator, startCloudflareTunnel, createEmulator } from "@mcpapps/dev";

const emulator = await serveEmulator(app, { port: 5179 });
console.log(emulator.url);     // http://localhost:5179
emulator.reload();             // push a live-reload to connected pages
```

- `serveEmulator(app, { port? })` → `{ url, port, close, reload }` — serves the emulator + the real `/mcp` endpoint on Node.
- `createEmulator(app, options?)` → `{ hono, reload }` — the underlying Hono app (mount it yourself).
- `startCloudflareTunnel({ port })` → `{ url, stop }` — an ephemeral public tunnel via `cloudflared`.

## What the emulator gives you

A real sandboxed iframe speaking the exact mcp-ui `postMessage` protocol, a mock tool-call loop that runs your real handlers, a theme toggle, and SSE-based live-reload with tool-result replay. See the [emulator guide](/guides/emulator).
