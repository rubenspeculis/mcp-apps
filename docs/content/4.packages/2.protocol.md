# @mcpapps/protocol

Pure TypeScript types and constants for the MCP Apps / mcp-ui contract. Zero runtime dependencies — the single source of truth every other package imports.

## Constants

```ts
import { PROTOCOL_VERSION, MCP_APP_MIME, HostMethods, AppNotifications, HostNotifications } from "@mcpapps/protocol";

PROTOCOL_VERSION;                 // "2026-01-26"
MCP_APP_MIME;                     // "text/html;profile=mcp-app"
HostMethods.Initialize;           // "ui/initialize"  (app → host requests)
HostMethods.CallTool;             // "tools/call"
HostMethods.Message;              // "ui/message"
AppNotifications.Initialized;     // "ui/notifications/initialized"  (app → host notifications)
AppNotifications.SizeChanged;     // "ui/notifications/size-changed"
HostNotifications.ToolResult;     // "ui/notifications/tool-result"  (host → app notifications)
HostNotifications.HostContextChanged; // "ui/notifications/host-context-changed"
```

Keeping every wire-level string in one place means a spec change is a one-line edit, never a hunt across packages.

## Key types

- `UiMeta` / `ToolMetaWithUi` — the `_meta.ui` binding: `resourceUri` + optional `csp`, `permissions`, `domain`, `prefersBorder`.
- `McpUiResourceCsp` — `{ connectDomains?, resourceDomains?, frameDomains?, baseUriDomains? }`; `McpUiPermissions` — camera/microphone/geolocation/clipboardWrite.
- `InitializeParams` / `InitializeResult` / `HostContext` — the `ui/initialize` handshake payloads (theme, `containerDimensions`, `toolInfo`, styles).
- `SizeChangedParams`, `ToolInputParams`, `UiMessageParams`.
- `ToolResultEnvelope<T>` — `{ toolName, structuredContent: T, content?, isError? }`; `ThemeState` — `{ colorScheme, tokens }`.
- `CompiledComponent` — `{ uri, html, assets?, basePath?, htmlAsset?, csp?, permissions?, domain?, prefersBorder? }`, the artifact a renderer produces and the server serves. `htmlAsset` points at a static asset the server reads at request time (for large inlined Flutter).
- JSON-RPC message types + guards (`isJsonRpcRequest`, `isJsonRpcResponse`, …).
