# @mcpapps/typegen

Generates typed Dart models from `zod` tool schemas, using JSON Schema as the language-neutral interchange format.

## API

```ts
import { zodToDart, writeDartModels } from "@mcpapps/typegen";
import { getWeatherOutput } from "./schemas";

await writeDartModels(
  [{ name: "GetWeatherOutput", schema: getWeatherOutput }],
  "flutter/lib/generated/models.dart",
);
```

- `zodToDart(specs)` → Dart source (header + one class tree per spec).
- `writeDartModels(specs, outFile)` — generate and write the `.dart` file.
- `jsonSchemaToDart(name, schema)` — the lower-level JSON Schema → Dart emitter.

## Output

For an object schema with nested objects and lists, it emits classes with `fromJson`/`toJson`, nested item classes, and nullable fields for optional properties:

```dart
class GetWeatherOutput {
  final double tempC;
  final String condition;
  final List<GetWeatherOutputHourlyItem> hourly;
  factory GetWeatherOutput.fromJson(Map<String, dynamic> j) => /* … */;
  Map<String, dynamic> toJson() => /* … */;
}
```

JSON Schema as the interchange format means future renderers (Swift, Kotlin) can reuse the same pipeline. See [type safety](/concepts/type-safety).
