# @mcpapps/cli

The scaffolder and the project CLI.

## `@mcpapps/create`

```bash [Terminal]
pnpm create @mcpapps my-app [--renderer vue|flutter] [--name <name>] [--force]
```

Scaffolds a complete, runnable project for the chosen renderer. Dependency specs auto-resolve to `workspace:*` inside the monorepo or published versions outside it.

## `mcpapps`

Run inside a project with an `mcpapps.config.ts`:

| Command | Does |
| --- | --- |
| `mcpapps dev` | Compile components, serve the emulator, live-reload. |
| `mcpapps serve [--stdio]` | Run the MCP server (HTTP or stdio). |
| `mcpapps build` | Compile components. |
| `mcpapps deploy` | Compile, then `wrangler deploy`. |

The CLI loads your project's `app` and its `@mcpapps/*` packages at runtime (via `jiti`), keeping its own dependency footprint small.

## Programmatic

```ts
import { createApp, defineConfig } from "@mcpapps/cli";
```

- `defineConfig(config)` — type-safe `mcpapps.config.ts` helper.
- `createApp(options)` — the scaffolder's programmatic entry.
