# Packages

@mcpapps is a small set of focused packages. The dependency direction is acyclic: `protocol` is consumed by everything; renderers consume `client-core`; the server consumes a renderer's *build output* (an HTML string), never its runtime.

| Package | Responsibility |
| --- | --- |
| [`@mcpapps/protocol`](/packages/protocol) | Types + constants for the MCP Apps / mcp-ui contract. Single source of truth. |
| [`@mcpapps/client-core`](/packages/client-core) | The in-iframe runtime: the only `postMessage` JSON-RPC peer. |
| [`@mcpapps/server`](/packages/server) | Hono MCP server: `defineApp`/`defineTool`, `ui://` serving, stdio. |
| [`@mcpapps/vue`](/packages/vue) | Vue hooks + renderer adapter. |
| [`@mcpapps/vite-plugin-vue`](/packages/vite-plugin-vue) | Compile a Vue SFC into a `ui://` resource. |
| [`@mcpapps/flutter`](/packages/flutter) | Compile a Flutter Web app into a `ui://` component. |
| [`mcpapps_bridge`](/packages/flutter-bridge) | Dart ↔ host bridge (`runMcpApp`, `McpApp.of`). |
| [`@mcpapps/dev`](/packages/dev) | Host emulator + Cloudflare tunnel. |
| [`@mcpapps/cli`](/packages/cli) | `create-mcpapp` + the `mcpapps` CLI. |
| [`@mcpapps/typegen`](/packages/typegen) | zod → JSON Schema → Dart models. |
