# @mcpapps/server

A fetch-native MCP server: `defineApp`/`defineTool`, `ui://` resource serving, and transports for HTTP (Hono) and stdio.

## API

```ts
import { defineApp, defineTool, createMcpHandler } from "@mcpapps/server";
import { mountMcp } from "@mcpapps/server/hono";
import { serveStdio } from "@mcpapps/server/stdio";
```

- `defineTool({ name, description?, inputSchema, outputSchema, ui?, handler })` — typed tool; args/return inferred from the `zod` schemas.
- `defineApp({ name, version, renderer, tools, compat? })` → `McpApp`.
- `createMcpHandler(app, { cors?, resolveHtml? })` → `(request: Request) => Promise<Response>` — the Workers-native core. `resolveHtml(component, request)` resolves a component's HTML at `resources/read` time (e.g. from a static asset).
- `mountMcp(hono, app, { path? })` — mount on a Hono instance (Node + Workers). Its default `resolveHtml` reads `component.htmlAsset` from the Worker's `ASSETS` binding, so large inlined components are served from a static asset instead of bloating the script.
- `serveStdio(app)` — newline-delimited JSON-RPC on stdin/stdout.

## Behaviour

- Emits `_meta.ui` on `tools/list`, `resources/list`, and `resources/read`: `resourceUri` plus any `csp`/`permissions`/`domain`/`prefersBorder` the component declares (and the `openai/widgetCSP` mirror when `compat: true`).
- Serves `ui://` HTML via `resources/read` — from `component.html`, or via `resolveHtml`/`htmlAsset` for large inlined documents.
- Validates tool args + output against the schemas; emits `structuredContent`.
- Implements MCP over Streamable HTTP.

See the [server guide](/guides/the-server) for full examples.
