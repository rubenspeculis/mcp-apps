# @mcpapps/server

A fetch-native MCP server: `defineApp`/`defineTool`, `ui://` resource serving, and transports for HTTP (Hono) and stdio.

## API

```ts
import { defineApp, defineTool, createMcpHandler } from "@mcpapps/server";
import { mountMcp } from "@mcpapps/server/hono";
import { serveStdio } from "@mcpapps/server/stdio";
```

- `defineTool({ name, description?, inputSchema, outputSchema, ui?, handler })` — typed tool; args/return inferred from the `zod` schemas.
- `defineApp({ name, version, renderer, tools, compat? })` → `McpApp`.
- `createMcpHandler(app, options?)` → `(request: Request) => Promise<Response>` — the Workers-native core.
- `mountMcp(hono, app, { path? })` — mount on a Hono instance (Node + Workers).
- `serveStdio(app)` — newline-delimited JSON-RPC on stdin/stdout.

## Behaviour

- Writes `_meta.ui.resourceUri` (and vendor mirror keys when `compat: true`).
- Serves `ui://` HTML via `resources/read`.
- Validates tool args + output against the schemas; emits `structuredContent`.
- Implements MCP over Streamable HTTP.

See the [server guide](/guides/the-server) for full examples.
