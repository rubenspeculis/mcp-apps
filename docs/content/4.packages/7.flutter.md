# @mcpapps/flutter

Build-time integration that compiles a Flutter Web app into a `ui://` MCP App component — either a multi-file bundle (`assets` + `basePath`, for the emulator) or a single self-contained HTML document (`inline: true`, for hosts like Claude).

## API

```ts
import { buildFlutterComponent, inlineFlutterBuild, resolveFlutterBin } from "@mcpapps/flutter";

const component = await buildFlutterComponent({
  projectDir: "/abs/path/flutter",
  uri: "ui://weather-flutter/get_weather",
  inline: true,            // one self-contained HTML (no sibling assets)
  // flutterBin auto-detected if omitted
});
```

- `buildFlutterComponent(options)` → `CompiledComponent`. Runs `flutter build web --no-web-resources-cdn` (CanvasKit bundled locally), includes the `mcpapps-host.js` glue (the `client-core` bridge), and always declares `fonts.gstatic.com` in `component.csp` (Flutter's runtime Noto font fallback). With `inline: true` it returns one `html` document and no `assets`/`basePath`; otherwise an `assets` map + `basePath`. Pass `csp`/`permissions`/`domain`/`prefersBorder` to extend the component's `_meta.ui`.
- `inlineFlutterBuild({ webDir, hostGlue })` → the self-contained HTML: inline scripts, a `fetch`/XHR shim serving `.wasm`/fonts from a base64 map, and a runtime import-map redirecting CanvasKit's dynamic `import()` to a `data:` module. Prunes to the chromium CanvasKit variant.
- `resolveFlutterBin(projectDir?)` — finds Flutter via `FLUTTER_BIN` → `PATH` → fvm (`.fvmrc`) → fvm default.

## How the bridge works

The document hosts the `client-core` glue; the Dart side ([`mcpapps_bridge`](/packages/flutter-bridge)) calls into it via `dart:js_interop`, exchanging JSON strings — so Dart talks to the host through the same bridge Vue uses.

## Serving the inlined document

The inlined HTML is ~10MB — too large for a Worker script bundle. Write it as a Cloudflare static asset and return it from `resources/read` via `CompiledComponent.htmlAsset` + the Worker `ASSETS` binding (`@mcpapps/server` `mountMcp` resolves it). See [Deploy to Cloudflare](/guides/deploy-cloudflare) and [renderers](/concepts/renderers).
