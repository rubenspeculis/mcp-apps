# @mcpapps/flutter

Build-time integration that compiles a Flutter Web app into a `ui://` MCP App component (loader HTML + cached assets).

## API

```ts
import { buildFlutterComponent, resolveFlutterBin } from "@mcpapps/flutter";

const component = await buildFlutterComponent({
  projectDir: "/abs/path/flutter",
  uri: "ui://weather-flutter/get_weather",
  // flutterBin auto-detected if omitted
});
```

- `buildFlutterComponent(options)` → `CompiledComponent` with `assets` + `basePath`. Runs `flutter build web`, injects the `mcpapps-host.js` glue (the `client-core` bridge), and packages the bundle.
- `resolveFlutterBin(projectDir?)` — finds Flutter via `FLUTTER_BIN` → `PATH` → fvm (`.fvmrc`) → fvm default.

## How the bridge works

The loader HTML hosts the `client-core` glue; the Dart side ([`mcpapps_bridge`](/packages/flutter-bridge)) calls into it via `dart:js_interop`, exchanging JSON strings — so Dart talks to the host through the same bridge Vue uses.

Because the bundle is multi-megabyte, a Flutter component is served as cached assets via iframe `src`, not inlined. See [renderers](/concepts/renderers).
