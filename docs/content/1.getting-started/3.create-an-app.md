# Create an app

The `@mcpapps/create` scaffolder generates a complete, runnable project — a tool with `zod` schemas, a component, the Hono server, a Cloudflare Workers entry, and the dev emulator — wired and ready.

## Scaffold

::code-group
```bash [Vue]
pnpm create @mcpapps my-app
```

```bash [Flutter]
pnpm create @mcpapps my-app --renderer flutter
```
::

Then:

```bash [Terminal]
cd my-app
pnpm install
pnpm dev          # Vue → host emulator on http://localhost:5179
# pnpm start      # Flutter → builds the web bundle, emulator on :5189
```

## Options

| Flag | Description |
| --- | --- |
| `--renderer <vue\|flutter>` | Which renderer to scaffold (default `vue`). |
| `--name <name>` | Package name (defaults to the directory name). |
| `--force` | Scaffold into a non-empty directory. |
| `--dep <spec>` | Override the `@mcpapps/*` dependency spec. |

::note
The scaffolder auto-detects whether it's running inside the `@mcpapps` monorepo. Inside, it wires `workspace:*` dependencies; outside, it pins published versions — so generated projects install in both contexts.
::

## What you get

A Vue project looks like:

```
my-app/
├── mcpapps.config.ts          # renderer + components + app entry
├── src/
│   ├── app.ts                 # defineApp + defineTool
│   ├── worker.ts              # Cloudflare Workers entry
│   ├── shared/schemas.ts      # zod input/output schemas
│   ├── mcpapps.d.ts           # augments the typed tool registry
│   └── components/
│       └── GreetingCard.vue   # the tool's UI
└── package.json               # dev / serve / build / deploy via the mcpapps CLI
```
