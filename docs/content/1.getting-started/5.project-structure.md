# Project structure

## `mcpapps.config.ts`

The config the CLI reads to compile components, load your app, and serve it:

```ts [mcpapps.config.ts]
import { defineConfig } from "@mcpapps/cli";

export default defineConfig({
  renderer: "vue",
  app: "./src/app.ts",                       // module exporting `app`
  generated: "./src/generated/components.ts", // where compiled components are written
  components: [
    {
      name: "weatherCard",
      entry: "./src/components/WeatherCard.vue",
      uri: "ui://weather-app/get_weather",
      title: "Weather",
    },
  ],
  port: 5179,
});
```

| Field | Purpose |
| --- | --- |
| `renderer` | `"vue"` or `"flutter"`. |
| `app` | Path to the module exporting your `defineApp` result. |
| `components` | Vue components to compile into `ui://` resources (name, entry, uri). |
| `generated` | Where the compiled-components module is written (imported by `app.ts`). |
| `port` | Default emulator port. |

## The CLI scripts

A scaffolded `package.json` delegates to the `mcpapps` CLI:

| Script | Runs |
| --- | --- |
| `pnpm dev` | `mcpapps dev` — emulator + live-reload |
| `pnpm serve` | `mcpapps serve` — MCP server over HTTP (`--stdio` for stdio) |
| `pnpm build` | `mcpapps build` — compile components |
| `pnpm deploy` | `mcpapps deploy` — compile + `wrangler deploy` |

See the [CLI reference](/packages/cli) for details.
