# Your first app

Let's build a `get_weather` tool whose result renders as a Vue card. Every piece is typed from one `zod` schema.

## 1. Define the schemas

```ts [src/shared/schemas.ts]
import { z } from "zod";

export const getWeatherInput = z.object({ city: z.string() });
export const getWeatherOutput = z.object({
  tempC: z.number(),
  condition: z.string(),
  hourly: z.array(z.object({ hour: z.number(), tempC: z.number() })),
});

export type ToolMap = {
  get_weather: {
    input: z.infer<typeof getWeatherInput>;
    output: z.infer<typeof getWeatherOutput>;
  };
};
```

## 2. Author the component

Hooks read the host bridge; types flow from the registry augmentation.

```vue [src/components/WeatherCard.vue]
<script setup lang="ts">
import { useCallTool, useToolResult } from "@mcpapps/vue";

const data = useToolResult<"get_weather">();      // typed: { tempC, condition, hourly } | null
const refresh = useCallTool("get_weather");        // typed args + result
</script>

<template>
  <div v-if="data">
    <h2>{{ Math.round(data.tempC) }}° — {{ data.condition }}</h2>
    <button @click="refresh({ city: 'London' })">Refresh</button>
  </div>
</template>
```

```ts [src/mcpapps.d.ts]
import type { ToolMap } from "./shared/schemas.js";

declare module "@mcpapps/vue" {
  interface ToolRegistry extends ToolMap {}
}
```

## 3. Define the tool + app

```ts [src/app.ts]
import { defineApp, defineTool } from "@mcpapps/server";
import { weatherCard } from "./generated/components.js"; // compiled by the CLI
import { getWeatherInput, getWeatherOutput } from "./shared/schemas.js";

const getWeather = defineTool({
  name: "get_weather",
  description: "Current weather for a city.",
  inputSchema: getWeatherInput,
  outputSchema: getWeatherOutput,
  ui: weatherCard,
  handler: ({ city }) => ({
    tempC: 21,
    condition: "Sunny",
    hourly: [{ hour: 9, tempC: 19 }, { hour: 12, tempC: 22 }],
  }),
});

export const app = defineApp({
  name: "weather-app",
  version: "1.0.0",
  renderer: "vue",
  compat: true,
  tools: [getWeather],
});
```

## 4. Run it

```bash [Terminal]
pnpm dev
```

Open the emulator, invoke `get_weather`, and the card renders inside a real sandboxed iframe. Click **Refresh** and the `useCallTool` round-trips back through the server, re-rendering with the new result.

::tip
The same `getWeatherOutput` schema validates the handler's return on the server, types `useToolResult` in Vue, and — for a Flutter app — generates a Dart `GetWeatherOutput` model. One source of truth.
::
