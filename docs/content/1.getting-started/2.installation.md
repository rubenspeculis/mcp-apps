# Installation

## Requirements

- **Node.js** ≥ 22
- **pnpm** (recommended) — the framework is developed as a pnpm workspace
- For the **Flutter** renderer: the Flutter SDK (auto-detected via `FLUTTER_BIN`, your `PATH`, or [fvm](https://fvm.app))
- For **deployment**: a Cloudflare account and [`wrangler`](https://developers.cloudflare.com/workers/wrangler/)

## The fastest path

You usually don't install packages by hand — scaffold a project instead:

```bash [Terminal]
pnpm create mcpapp my-app
cd my-app && pnpm install && pnpm dev
```

This wires up the right packages for your chosen renderer. See [Create an app](/getting-started/create-an-app).

## Adding to an existing project

A Vue MCP App typically depends on:

```bash [Terminal]
pnpm add @mcpapps/server @mcpapps/vue @mcpapps/client-core @mcpapps/protocol hono zod
pnpm add -D @mcpapps/cli @mcpapps/dev @mcpapps/vite-plugin-vue vite @vitejs/plugin-vue
```

A Flutter MCP App depends on `@mcpapps/server`, `@mcpapps/flutter`, `@mcpapps/client-core` and the `mcpapps_bridge` Dart package (added to your Flutter project's `pubspec.yaml`).

::tip
Every package is published under the `@mcpapps/*` scope. The `mcpapps_bridge` Dart package is consumed through `pubspec.yaml`, not npm.
::
