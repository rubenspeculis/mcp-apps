# Overview

**@mcpapps** is a less-opinionated, multi-renderer framework for building **MCP Apps** — interactive UIs returned from [Model Context Protocol](https://modelcontextprotocol.io) tools and rendered inside hosts like **Claude**, **ChatGPT** and **VS Code**.

It implements the standard MCP Apps / [mcp-ui](https://mcpui.dev) contract — a tool binds a `ui://` HTML resource that the host renders in a sandboxed iframe and talks to over JSON-RPC `postMessage` — and then gets out of your way.

## What is an MCP App?

The Model Context Protocol lets an AI host call **tools** on a server. An **MCP App** extends that: a tool can return not just data, but a small interactive UI. The host renders it inline, and the UI can call back into the server to fetch fresh data or trigger actions.

Concretely, the contract is:

1. A tool advertises a UI resource via `_meta.ui.resourceUri` (a `ui://…` URI).
2. The host reads that resource — an HTML document — and renders it in a **sandboxed iframe**.
3. The iframe and host exchange **JSON-RPC over `postMessage`**: the host pushes the tool result in; the app calls tools back out.

::note
@mcpapps follows this standard so your apps run in real hosts today. The framework's value is in *how comfortable* it makes building against that contract — and in not locking you into one UI framework, server, or runtime.
::

## How it fits together

- **`@mcpapps/server`** — define tools with `zod` schemas on a [Hono](https://hono.dev) server. Runs on Cloudflare Workers, Node, Bun and Deno from the same `fetch` handler.
- **`@mcpapps/client-core`** — the single runtime that lives inside the iframe and speaks the host protocol. Every renderer sits on top of it, so they never diverge.
- **A renderer, chosen per app** — `@mcpapps/vue` (+ its Vite plugin) or `@mcpapps/flutter` (+ the `mcpapps_bridge` Dart package).
- **`@mcpapps/dev`** — a host emulator and a Cloudflare tunnel for local development.
- **`@mcpapps/cli`** — `@mcpapps/create` to scaffold, and `mcpapps dev/serve/build/deploy` to run.
- **`@mcpapps/typegen`** — turns your `zod` schemas into typed Dart models.
