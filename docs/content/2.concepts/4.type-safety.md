# Type safety

A single `zod` schema per tool drives type safety across the whole stack.

```
            ┌─────────────────────────► server: handler args + output validated
zod schema ─┼─────────────────────────► Vue: useToolResult<"tool">() inferred
            └─── JSON Schema ─────────► Dart: generated fromJson/toJson model
```

## On the server

`defineTool` is generic over its input/output schemas, so the `handler` arguments and return type are inferred, and the runtime **validates** both against the schemas before the result reaches the host as `structuredContent`.

Object output schemas are strict: the emitted JSON Schema includes `additionalProperties: false`, and the server validates the handler result before returning it. Return only fields declared in the tool's `outputSchema`, or include an explicit catch-all/record field in the schema when extra keys are intentional.

## In Vue — inference, no codegen

The Vite plugin emits a typed registry; you augment `@mcpapps/vue`'s `ToolRegistry` interface so hooks resolve types at compile time:

```ts
const data = useToolResult<"get_weather">();   // Ref<{ tempC; condition; hourly } | null>
const call = useCallTool("get_weather");        // (args: { city: string }) => Promise<output>
```

Zero runtime cost — pure TypeScript inference.

## In Flutter — generated models

Dart can't consume TypeScript types, so `@mcpapps/typegen` converts each schema to **JSON Schema**, then emits Dart classes with `fromJson`/`toJson` (including nested objects and lists, with nullable handling for optional fields):

```dart
final data = GetWeatherOutput.fromJson(McpApp.of(context).result!);
Text('${data.tempC.round()}° ${data.condition}');
```

JSON Schema is the language-neutral interchange format, so future renderers (Swift, Kotlin) can reuse the same pipeline.

::tip
Supported `zod` subset: objects, strings, numbers/integers, booleans, arrays, nested objects, and optional fields. Constructs that don't cross cleanly to JSON Schema (transforms, refinements) are validated on the TypeScript side only.
::
