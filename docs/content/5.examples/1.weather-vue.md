# Weather (Vue)

The `weather-vue` example is a complete Vue MCP App: a `get_weather` tool whose result renders as a card with an hourly bar chart, plus a Refresh button that round-trips back through the server.

## Run it

```bash [Terminal]
pnpm --filter @mcpapps/example-weather-vue dev   # or: pnpm dev:vue
```

Open `http://localhost:5179`, invoke `get_weather`, and the card renders in a sandboxed iframe. Clicking **Refresh** calls `useCallTool("get_weather")`, which round-trips through the host → server → response and reactively re-renders.

## What it demonstrates

- A typed tool (`zod` schemas) bound to a compiled Vue component.
- `useToolResult<"get_weather">()` and `useCallTool` with full inference.
- The self-contained `ui://` HTML (Vue + `client-core` inlined) rendered in a real iframe.
- The full bidirectional contract: host → `tool-result` → render, and app → `tools/call` → re-render.

It also runs on Cloudflare Workers unchanged via its `worker.ts` entry — see [deploy](/guides/deploy-cloudflare).
