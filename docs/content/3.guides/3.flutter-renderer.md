# Flutter components

A Flutter MCP App component is a Flutter Web app that reads the host bridge through the `mcpapps_bridge` Dart package.

## Boot and read

```dart [lib/main.dart]
import 'package:flutter/material.dart';
import 'package:mcpapps_bridge/mcpapps_bridge.dart';
import 'generated/models.dart'; // typed model from your zod schema

void main() => runMcpApp(const WeatherApp());

class WeatherApp extends StatelessWidget {
  const WeatherApp({super.key});
  @override
  Widget build(BuildContext context) {
    final raw = McpApp.of(context).result;
    if (raw == null) return const Text('Waiting…');
    final data = GetWeatherOutput.fromJson(raw);
    return Text('${data.tempC.round()}° ${data.condition}');
  }
}
```

| API | Purpose |
| --- | --- |
| `runMcpApp(widget)` | Boots the engine, wires the bridge, signals `ready()`. |
| `McpApp.of(context)` | The controller: `.result`, `.colorScheme`, `.callTool(name, args)`. |
| `GetWeatherOutput.fromJson` | Typed model generated by [`@mcpapps/typegen`](/packages/typegen). |

## How it connects

`@mcpapps/flutter` runs `flutter build web`, then wraps the output in a loader HTML that also includes a small `mcpapps-host.js` glue (the `client-core` bridge). The Dart side calls into that glue via `dart:js_interop`, exchanging JSON strings — so Dart talks to the host through the **same bridge** Vue uses.

## Requirements & trade-offs

- Needs the Flutter SDK (auto-detected via `FLUTTER_BIN`, your `PATH`, or fvm).
- The bundle is multi-megabyte (CanvasKit); served as cached assets via iframe `src`. See [renderers](/concepts/renderers) for the honest trade-offs.

## Run it

```bash [Terminal]
pnpm start    # builds the Flutter web bundle and serves the emulator on :5189
```
