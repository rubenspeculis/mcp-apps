# stdio & the CLI

## stdio transport

Desktop hosts that spawn a server subprocess speak MCP over **stdio** (newline-delimited JSON-RPC on stdin/stdout). Serve it with:

```bash [Terminal]
mcpapps serve --stdio
```

Or programmatically:

```ts
import { serveStdio } from "@mcpapps/server/stdio";
import { app } from "./app";

serveStdio(app);
```

Both transports share the same message processor, so behavior is identical to the HTTP server.

## The `mcpapps` CLI

Run inside a project that has an `mcpapps.config.ts`:

| Command | Does |
| --- | --- |
| `mcpapps dev` | Compile components, serve the emulator, live-reload on edits. |
| `mcpapps serve [--stdio]` | Run the MCP server over HTTP (or stdio). |
| `mcpapps build` | Compile components. |
| `mcpapps audit [--static-only \| --runtime]` | Validate the app conforms to the spec. |
| `mcpapps deploy` | Compile, then `wrangler deploy`. |

The CLI loads your project's `app` and its `@mcpapps/*` packages at runtime, so its own footprint stays small. See the [CLI reference](/packages/cli).

## Auditing your app

`mcpapps audit` is a conformance check you run before publishing — the equivalent
of a preflight lint for an MCP App. It runs in two layers:

**Static** (always): validates `mcpapps.config.ts` (or, for Flutter, `src/worker.ts`),
checks every `ui://` URI and component entry, the tool definitions (name format,
input/output schemas), and that each `csp` block only uses the supported buckets
(`connectDomains` / `resourceDomains` / `frameDomains` / `baseUriDomains`).

**Runtime** (default; `--static-only` to skip): compiles the components, serves the
[emulator](/guides/emulator), and drives it in a **headless browser** to assert each
UI tool completes the `ui/initialize` handshake, reports a **non-zero size** (the classic
0-height iframe collapse), and only contacts origins it declared in its CSP.

```bash [Terminal]
mcpapps audit                # static + runtime
mcpapps audit --static-only  # no browser (CI lint, fast)
```

The runtime layer needs Playwright (kept out of the CLI's install footprint):

```bash [Terminal]
pnpm add -D playwright && npx playwright install chromium
```

`audit` exits non-zero on any failure, so it drops straight into CI. Flutter's runtime
pass needs a prior `pnpm build`/`deploy` (its build needs the Flutter toolchain); without
one, the runtime check is skipped with a note and the static checks still run.
