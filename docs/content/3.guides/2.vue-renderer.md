# Vue components

A Vue MCP App component is a normal `.vue` SFC that reads the host bridge through three hooks.

## Hooks

```vue
<script setup lang="ts">
import { useToolResult, useCallTool, useTheme } from "@mcpapps/vue";

const data = useToolResult<"get_weather">();   // reactive Ref<output | null>, replayed on mount
const refresh = useCallTool("get_weather");      // (args) => Promise<output>
const theme = useTheme();                        // reactive { colorScheme, tokens }
</script>
```

| Hook | Returns |
| --- | --- |
| `useToolResult<"tool">()` | A `Ref` of the latest tool result's `structuredContent`, typed from the registry. |
| `useCallTool("tool")` | A typed function that invokes a tool and resolves its output. |
| `useTheme()` | A `Ref` of the host theme (light/dark + tokens). |

## Compilation

The `@mcpapps/vite-plugin-vue` plugin compiles the SFC — with the Vue runtime and `client-core` — into a single self-contained `ui://` HTML resource, and emits a typed registry so `useToolResult<"tool">()` infers the right type. The `mcpapps` CLI runs this for you on `dev`/`build`; you can also call `writeComponentsModule()` directly.

## Typing

Augment the registry once so the hooks are fully typed:

```ts [src/mcpapps.d.ts]
import type { ToolMap } from "./shared/schemas.js";
declare module "@mcpapps/vue" {
  interface ToolRegistry extends ToolMap {}
}
```

See [type safety](/concepts/type-safety) for how the schema flows through.
