# The Hono server

`@mcpapps/server` is a **fetch-native** MCP server. The core is a single `fetch(request) → Response` handler, so the same code runs on Cloudflare Workers, Node, Bun and Deno.

## Define tools and an app

```ts [src/app.ts]
import { defineApp, defineTool } from "@mcpapps/server";
import { z } from "zod";
import { weatherCard } from "./generated/components.js";

const getWeather = defineTool({
  name: "get_weather",
  description: "Current weather for a city.",
  inputSchema: z.object({ city: z.string() }),
  outputSchema: z.object({ tempC: z.number(), condition: z.string() }),
  ui: weatherCard,                       // binds _meta.ui.resourceUri
  handler: ({ city }) => ({ tempC: 21, condition: "Sunny" }),
});

export const app = defineApp({
  name: "weather-app",
  version: "1.0.0",
  renderer: "vue",
  compat: true,                          // mirror vendor _meta keys for cross-host support
  tools: [getWeather],
});
```

## Mount on Hono

The handler is pure Fetch API, so mounting is identical across runtimes:

::code-group
```ts [Cloudflare Workers]
import { Hono } from "hono";
import { mountMcp } from "@mcpapps/server/hono";
import { app } from "./app";

const hono = new Hono();
mountMcp(hono, app);          // mounts /mcp
export default hono;
```

```ts [Node]
import { serve } from "@hono/node-server";
import { Hono } from "hono";
import { mountMcp } from "@mcpapps/server/hono";
import { app } from "./app";

const hono = new Hono();
mountMcp(hono, app);
serve({ fetch: hono.fetch, port: 8787 });
```
::

## What the server does

- Registers each tool and writes normalized `_meta.ui.resourceUri` (plus vendor mirror keys when `compat: true`).
- Serves the compiled `ui://` HTML via `resources/read`.
- **Validates** tool arguments and output against the `zod` schemas, returning `structuredContent` the host forwards to the iframe.
- Implements MCP over Streamable HTTP. For desktop hosts that spawn a process, a **stdio** transport is available — see [stdio & the CLI](/guides/stdio-and-cli).

::note
`mountMcp` works the same on Workers and Node because Hono and the handler are built on the Web Fetch API. There is no Express, and no Node-only assumption in the request path.
::
