# The dev emulator

`mcpapps dev` serves a **host emulator** — a local page that stands in for Claude/ChatGPT while you build.

```bash [Terminal]
pnpm dev      # → http://localhost:5179
```

## What it does

- Renders your component in a **real sandboxed iframe** with the same `sandbox`/CSP a host applies.
- Speaks the **exact mcp-ui `postMessage` JSON-RPC** — your component can't tell the emulator from a real host.
- Runs a **mock tool-call loop**: invoking a tool runs your *real* `handler`; the result is pushed to the iframe as `ui/notifications/tool-result`, and the app's `tools/call` requests are routed back through the handler.
- Provides a theme toggle and generates sample arguments from your `zod` input schema.

## Live-reload

Edit a component and save: the CLI recompiles it and pushes a reload over Server-Sent Events. The emulator re-renders and **replays the last tool result**, so your edit appears immediately without re-invoking the tool or losing state — the component is driven by the tool result, which `client-core` caches.

::tip
The emulator drives the same `/mcp` endpoint a real host would, so what you see locally is what the host renders. To go one step further and test inside an actual host, expose the dev server with the [Cloudflare tunnel](/guides/tunnel).
::
